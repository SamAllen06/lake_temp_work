# Combinatorial Fault Finding Runs

This document provides the results for runs of the Model Testing Framework on
LakeTemperature via SPEL. These results used the netCDF cases plugin, in combination 
with test case indices generated by ACTS and a partition using linear interpolation
on ranges provided by our collaborators. Checks were based on rules we implemented based
on rules provided by our collaborators. These runs were completed using commit
`880fd2665d4189f611d9592f16ded7099ba86021`.

## Sampling

The samples used for these runs were combinatorial cases. In order to generate them, we
used [ACTS](https://csrc.nist.gov/Projects/automated-combinatorial-testing-for-software/downloadable-tools)
to generate indices for each model constant. For floating point and integer constants,
we used eleven values, for boolean constants we just used zero and one. The
configuration we used with ACTS is `/test_resources/acts/lake_temp_combinatorial.conf`.
The degree of interation (DOI) referenced in the runs below represent the degree of 
interaction we instructed ACTS to use. For these tests, we used linear interpolation
between physical ranges provided to us by our collaborators in
`/documentation/from_collaborators/2025/12/LakeTemperatureInput.pdf`. The ranges are
listed below, and are also defined in `/test_resources/acts/lake_temp_partition.nc`. 
This file was combined with the CSV files generated by ACTS using the `csv2nc` script
to generate the cases that were ultimately stored as
`/testing/lake_temp/plugin_inputs/netcdf_cases/lake_temp_i{2..5}.nc` with the number at
the end of the file representing the degree of interaction.

## Analysis

Analysis was performed by the Fault Finding plugin using the checks in 
`/testing/lake_temp/plugin_inputs/fault_checks/`. These checks were based off of simple
checks described in `/documentation/from_collaborators/2025/12/LaketempatureOutput.pdf`
and more complicated physical law checks based on
`/documentation/from_collaborators/2025/12/LakeTemperatureTestPlan.pdf`. Checks in the
second file often had preconditions, and when the check's preconditions were not met, it
was simply skipped.
